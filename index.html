Structure a document with HTML

Boxes

When you look at a webpage it’s important to be able to see it as one big box containing tons of small boxes and your ability to identify these boxes will make you get a better structure of your html document and as a result get a better webpage. These boxes are represented as divs.

Tree Structure

the previous concept of boxes gives rise to a hierarchical relationship between the elements where the main box can be seen as parents and the small boxes inside would be child of that element.

Developer Tools

The developer tool is a powerful instrument as it allows you to dissect the page. Through the dev tools you are able to see both the sturcture and the style of each element on the webpage.

The basics of CSS

Definitions

CSS stands for cascading style sheet. Cascading simply means that the style that the browser will look for the most specific rule (style) to apply to an element. It goes from the bigger style (given to the parent) and makes its way down the tree until it finds a more specific rule (if any)

Some CSS rules

There are some cool and powerful rules we can use such as:
<display: flex > which will help you position your boxes properly within the same div.
<border radius:xx> which will give your box rounded corners if that’s what you are going for
